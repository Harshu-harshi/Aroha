AI Recipe Generator

Complete Project Documentation

1. PROJECT OVERVIEW

The AI Recipe Generator is a Django web application that:

  • Generates custom recipes using OpenAI's GPT-3.5-turbo model
  • Allows users to input available ingredients
  • Provides filters for dietary preferences, cuisine types, and meal types
  • Stores recipe history in a database
  • Features a responsive Bootstrap interface

2. TECHNICAL SPECIFICATIONS

Component Technology
Framework Django 5.1+
Database SQLite (default)
Frontend HTML5, Bootstrap 5.3
API OpenAI GPT-3.5-turbo
Python 3.8+

4. INSTALLATION GUIDE

STEP 1: Prerequisites

  • Python 3.8+ installed
  • OpenAI API account
  • Git (optional)

STEP 2: Setup

  1. Clone repository (if applicable)
  2. Create and activate virtual environment: 
    # Windows
python -m venv venv && venv\Scripts\activate

# Mac/Linux
python3 -m venv venv && source venv/bin/activate
  3. Install dependencies: 
    pip install django openai python-dotenv
  4. Create .env file in project root: 
    OPENAI_API_KEY=your_api_key_here
  5. Run migrations: 
    python manage.py migrate
  6. Start development server: 
    python manage.py runserver

5. DETAILED COMPONENT DOCUMENTATION

5.1 MODELS (models.py)

class GeneratedRecipe(models.Model):
    ingredients = models.TextField()  # user-provided ingredients
    recipe = models.TextField()       # AI-generated recipe content
    created_at = models.DateTimeField(auto_now_add=True)  # auto-generated timestamp

5.2 VIEWS (views.py)

Key Functions:

1. get_openai_client()

  • Initializes and verifies OpenAI client
  • Validates API key
  • Handles authentication errors

2. generate_recipe_prompt()

  • Constructs detailed prompt for OpenAI
  • Incorporates user inputs:
    • Ingredients
    • Dietary preferences
    • Cuisine type
    • Meal type
  • Specifies output format

3. index()

  • Main view for recipe generation
  • Handles form submission
  • Calls OpenAI API
  • Saves successful generations
  • Implements error handling

4. recipe_history()

  • Displays all generated recipes
  • Orders by creation date (newest first)

5.3 TEMPLATES

1. base.html

  • Base template structure
  • Includes Bootstrap CSS/JS
  • Defines content block

2. index.html

  • Main input form with:
    • Ingredients textarea (required)
    • Dietary preferences dropdown
    • Cuisine type dropdown
    • Meal type dropdown
  • Form validation
  • Error message display

3. recipe_result.html

  • Displays generated recipe
  • Shows original ingredients
  • Provides navigation buttons

4. recipe_history.html

  • Lists all generated recipes
  • Shows creation timestamp
  • Displays ingredients and recipe
  • Empty state handling

8. USAGE INSTRUCTIONS

  1. Access the application at http://localhost:8000
  2. On the main page:
    • Enter available ingredients (comma separated)
    • Select optional filters:
      • Dietary preferences
      • Cuisine type
      • Meal type
  3. Click "Generate Recipe"
  4. View results:
    • Recipe name
    • Description
    • Ingredients list
    • Step-by-step instructions
    • Cooking time
    • Difficulty level
    • Serving size
  5. Options:
    • Generate another recipe
    • View recipe history

9. TROUBLESHOOTING

Common Issues:

1. OpenAI API errors:

  • Verify API key in .env
  • Check internet connection
  • Validate OpenAI account status

2. Form validation errors:

  • Ensure at least one ingredient is provided
  • Check for special characters that might cause issues

3. Database issues:

  • Run migrations if getting database errors
  • Verify SQLite file permissions

10. DEPLOYMENT GUIDE

Production Checklist:

  1. Set DEBUG = False in settings.py
  2. Configure ALLOWED_HOSTS
  3. Set up production database (PostgreSQL recommended)
  4. Configure static files serving
  5. Set up proper secret key
  6. Consider implementing:
    • Rate limiting
    • Caching
    • Monitoring
